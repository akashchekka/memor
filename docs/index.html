How Memor works

An LSM-tree-inspired architecture designed for instant writes and ranked reads.

01

Write

AI tools append memories as JSONL to memory.wal — fast, append-only, crash-safe. No coordination needed.

02

Compact

Merges WAL into memory.db, deduplicates via SHA-256, scores by relevance, enforces token budget.

03

Read

memor context retrieves the most relevant memories within budget — trigram index + BM25 ranking, sub-millisecond.

Four types of memory

Each type has its own lifecycle, compaction weight, and decay behavior.

@s

Semantic

Facts, decisions, architecture choices. The backbone of project knowledge.

@s #auth: OAuth2+PKCE via Auth0

Episodic

@e

Events and bugs fixed. Decays over time — lowest compaction weight, archived after 90 days.

@e #fix: N+1 query in dashboard loader
@p

Procedural

Commands, workflows, how-tos. The recipes your AI needs to execute tasks correctly.

@p #deploy: pnpm turbo deploy --filter=api
@f

Preference

Style conventions and developer preferences. Permanent — never decays or gets archived.

@f #ts: No any, use unknown + type guards

Full CLI, zero friction

Everything runs locally. No daemon, no background processes, no git commits.

memor init Set up .memor/ with hooks and skill files
memor add Append a new memory to the WAL
memor context Get ranked context within token budget
memor search <query> Full-text search with trigram + BM25
memor compact Merge WAL into snapshot, deduplicate
memor stats Entry counts, token usage, index health
memor reinforce <id> Boost relevance of a useful memory
memor knowledge add Index a document into the knowledge base
